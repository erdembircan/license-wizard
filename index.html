Features

Everything the LICENSE file needs

One predictable flow that guides a human and drives cleanly from a script.

Full SPDX catalog

Search and pick from the complete, up-to-date SPDX License List with type-ahead autocomplete.

Canonical text, every time

License text comes straight from the official SPDX data — no hand-copied variations creeping in.

Copyright customization

For licenses with fillable fields, keep the official text verbatim or fill in the holder, year, and the rest.

Manifest-aware

Reads the license declared in your package.json or composer.json and writes your choice back to every one it finds.

Remembers your choice

Optionally saves a small config file so re-runs start exactly where you left off.

Zero config to start

Run it with npx, answer a few prompts, done. No setup, no flags required.

How it works

A short, guided flow

Run it from the root of the project you want to license.

  1. 1

    Choose a license

    Start typing to search the SPDX catalog. If your project already declares one, it's offered as the default.

  2. 2

    Standard or customized

    Keep the official text verbatim, or fill in fillable fields like the copyright holder and year.

  3. 3

    Save your settings

    Choose whether to remember your selection — then the LICENSE file is written and your manifests updated.

$ npm install -g license-wizard
$ pnpm dlx license-wizard
$ yarn dlx license-wizard

Writes LICENSE to the current directory and records the SPDX id in every manifest present.

Scripts · CI · AI agents

The same flow,
without the prompts

Pass --license, --set, or --get-tokens and License Wizard runs as a single command. Miss a required field and it writes nothing, lists what's missing, and exits non-zero — so a calling agent knows exactly what to fix.

Non-interactive docs

# one-shot, fully scripted

$ npx license-wizard \

--license MIT \

--set "year=2026" \

--save-rc

◆ Wrote LICENSE (MIT)

◆ Saved .licensewizardrc.json

# keep it honest in CI

$ npx license-wizard --verify --strict

✓ LICENSE & manifests match the saved config

Reference

Available flags

Run license-wizard --help to print the same list from the CLI.

Flag What it does
--helpShow the help message and exit.
--verifyCheck the LICENSE file and every manifest's license field against the saved config, reconciling drift. Standalone mode.
--strictWith --verify, fail on any drift instead of reconciling — for CI.
--license <spdx-id>Select a license by SPDX identifier and run non-interactively.
--set <field=value>Set a copyright field for the chosen license (repeatable). Implies non-interactive mode.
--save-rcSave the resolved config to .licensewizardrc.json.
--save-npmSave to the "license-wizard" field of package.json.
--save-composerSave to the "license-wizard" field of composer.json.
--get-tokensList the copyright fields the selected license accepts and exit.
--dry-runPreview the license and skip every write.
