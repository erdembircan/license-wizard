# License File Conventions

A LICENSE file is the legal instrument that grants others permission to use your code. Unlike the `license` field in a package manifest, the LICENSE file contains the actual legal terms, including the copyright notice and the full license text.

Getting the file right involves three practical concerns: naming it correctly so platforms detect it, placing it in the right location, and filling in the copyright line without altering the standardized legal text. These are straightforward but frequently done wrong.

This section covers the conventions observed across the open-source ecosystem, from file naming to copyright line formatting, with examples from major projects.
