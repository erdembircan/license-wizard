# License Types

Open-source licenses fall along a spectrum from fully permissive to strongly copyleft. The core distinction is what obligations they place on people who use, modify, or distribute the licensed code.

**Permissive licenses** (MIT, Apache 2.0, BSD, ISC) allow virtually unrestricted use, including in proprietary software. Their primary condition is preserving the copyright notice and license text. They are the dominant choice in the npm and JavaScript ecosystems.

**Copyleft licenses** (GPL, LGPL, AGPL) require that derivative works be distributed under the same license. This ensures that modifications remain open-source, but it also restricts how the code can be incorporated into proprietary projects.

Each license defines specific **permissions** (what you can do), **conditions** (what you must do), and **limitations** (what you cannot claim). Understanding these three dimensions is essential for choosing the right license.
